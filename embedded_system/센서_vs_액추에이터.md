# 센서 vs 액추에이터

## 1. 수행 목표

본 문서는 임베디드 시스템에서 입력장치에 해당하는 센서와 출력장치에 해당하는 액추에이터의 종류와 역할을 정리하기 위해 작성하였다.
또한 센서와 액추에이터를 조합하여 만들 수 있는 실제 임베디드 장비를 조사하고, 주변에서 쉽게 찾을 수 있는 장비의 센서와 액추에이터 구성을 분석한다.

---

## 2. 센서의 정의

센서(Sensor)는 주변 환경이나 물리적인 상태를 감지하여 전기적인 신호로 바꾸는 장치이다.
컴퓨터에서 키보드, 마우스, 카메라가 입력장치 역할을 하듯이, 임베디드 시스템에서는 센서가 입력장치 역할을 한다.

예를 들어 온도 센서는 온도를 전압 또는 디지털 데이터로 변환하고, 조도 센서는 빛의 세기를 전기 신호로 변환한다.
마이크로컨트롤러는 이 센서값을 읽고, 조건에 따라 액추에이터를 제어한다.

```text
외부 환경 변화
→ 센서 감지
→ 전기 신호 변환
→ 마이크로컨트롤러 입력
→ 판단 및 제어
```

---

## 3. 센서의 기본 역할

| 역할          | 설명                                |
| ----------- | --------------------------------- |
| 환경 감지       | 온도, 습도, 빛, 거리, 움직임 등을 측정한다.       |
| 상태 확인       | 문 열림, 물체 접근, 물 부족, 가스 누출 등을 확인한다. |
| 데이터 입력      | 측정값을 아날로그 또는 디지털 데이터로 변환한다.       |
| 자동 제어 기준 제공 | 센서값을 기준으로 LED, 모터, 릴레이 등을 제어한다.   |

센서는 임베디드 시스템이 주변 상황을 인식할 수 있도록 해주는 장치이다.

---

## 4. 아날로그 센서와 디지털 센서

센서는 출력 방식에 따라 크게 아날로그 센서와 디지털 센서로 나눌 수 있다.

---

## 4.1 아날로그 센서

아날로그 센서는 측정값에 따라 연속적인 전압값을 출력한다.
아두이노는 이 전압값을 ADC를 통해 0~1023 사이의 디지털 수치로 변환한다.

예를 들어 5V 기준 아두이노에서는 다음과 같이 변환된다.

```text
0V   → 0
2.5V → 약 512
5V   → 1023
```

### 아날로그 센서 예시

| 센서         | 측정 대상  | 출력 형태 |
| ---------- | ------ | ----- |
| 조도 센서      | 빛의 세기  | 전압 변화 |
| 가변저항       | 회전 위치  | 전압 변화 |
| 토양 수분 센서   | 흙의 수분량 | 전압 변화 |
| LM35 온도 센서 | 온도     | 전압 변화 |
| 가스 센서      | 가스 농도  | 전압 변화 |

아날로그 센서는 회로가 단순하고 사용하기 쉽지만, 노이즈의 영향을 받을 수 있다.

---

## 4.2 디지털 센서

디지털 센서는 처음부터 디지털 신호를 출력하거나, 내부에서 측정값을 처리한 뒤 통신 방식으로 데이터를 전달한다.

### 디지털 센서 예시

| 센서          | 측정 대상    | 출력 방식               |
| ----------- | -------- | ------------------- |
| DHT11/DHT22 | 온도, 습도   | 디지털 데이터             |
| DS18B20     | 온도       | 1-Wire 통신           |
| HC-SR04     | 거리       | Trigger/Echo 디지털 펄스 |
| PIR 센서      | 사람 움직임   | HIGH/LOW 신호         |
| IMU 센서      | 가속도, 각속도 | I2C/SPI 통신          |
| 디지털 도어 센서   | 문 열림/닫힘  | HIGH/LOW 신호         |

디지털 센서는 값이 비교적 명확하고 통신을 통해 데이터를 주고받을 수 있지만, 센서마다 통신 규격과 라이브러리가 필요할 수 있다.

---

## 5. 아날로그 센서값을 디지털 수치로 바꾸는 방식

아두이노의 아날로그 입력 핀은 ADC(Analog to Digital Converter)를 사용한다.
ADC는 연속적인 전압값을 정해진 범위의 숫자로 바꿔준다.

아두이노 UNO의 경우 일반적으로 10-bit ADC를 사용하므로 0~1023 범위의 값을 얻는다.

```text
디지털 값 = 입력 전압 / 기준 전압 × 1023
```

예를 들어 기준 전압이 5V이고 센서 출력 전압이 2V라면 다음과 같이 계산할 수 있다.

```text
디지털 값 = 2V / 5V × 1023 ≒ 409
```

즉, 아날로그 센서는 실제 물리량을 전압으로 바꾸고, 아두이노는 이 전압을 숫자로 변환하여 처리한다.

---

## 6. 물리 값을 측정하는 센서 종류

| 분류     | 센서 예시                | 측정값    |
| ------ | -------------------- | ------ |
| 온도 센서  | LM35, TMP36, DS18B20 | 온도     |
| 습도 센서  | DHT11, DHT22         | 습도     |
| 조도 센서  | CDS, LDR             | 빛의 세기  |
| 거리 센서  | HC-SR04, 적외선 거리 센서   | 거리     |
| 움직임 센서 | PIR 센서               | 사람 움직임 |
| 가속도 센서 | MPU6050, ADXL345     | 가속도    |
| 자이로 센서 | MPU6050              | 각속도    |
| 압력 센서  | BMP280               | 대기압    |
| 수분 센서  | 토양 수분 센서             | 토양 수분량 |
| 빗물 센서  | Rain Sensor          | 빗물 감지  |
| 가스 센서  | MQ 계열                | 가스 농도  |
| 소리 센서  | 마이크 모듈               | 소리 크기  |
| 색상 센서  | TCS3200              | 색상     |
| 터치 센서  | 정전식 터치 센서            | 접촉 여부  |

이처럼 센서는 임베디드 시스템이 주변의 물리적인 상태를 감지하는 입력장치이다.

---

## 7. 액추에이터의 정의

액추에이터(Actuator)는 전기 신호를 받아 실제 물리적인 동작이나 출력으로 변환하는 장치이다.
컴퓨터에서 모니터, 스피커, 프린터가 출력장치 역할을 하듯이, 임베디드 시스템에서는 액추에이터가 출력장치 역할을 한다.

액추에이터는 전기를 이용하여 다음과 같은 출력을 만든다.

| 출력 형태 | 예시                    |
| ----- | --------------------- |
| 빛     | LED, RGB LED, 조명      |
| 소리    | 부저, 스피커               |
| 움직임   | DC 모터, 서보 모터, 스텝 모터   |
| 힘     | 솔레노이드, 전자석            |
| 열     | 히터                    |
| 유체 제어 | 펌프, 밸브                |
| 화면 표시 | LCD, OLED, LED Matrix |

따라서 액추에이터는 **전기로 출력값을 표현할 수 있는 모든 장치**라고 볼 수 있다.

---

## 8. 빛을 이용하는 액추에이터

빛을 출력하는 장치는 가장 기본적인 액추에이터이다.

| 장치           | 설명                     | 사용 예              |
| ------------ | ---------------------- | ----------------- |
| LED          | 전류가 흐르면 빛을 내는 소자       | 상태 표시등            |
| RGB LED      | 빨강, 초록, 파랑을 조합하여 색상 표현 | 경고등, 무드등          |
| NeoPixel LED | 개별 제어 가능한 RGB LED      | LED 스트립, 조명 효과    |
| 7-Segment    | 숫자를 표시하는 LED 장치        | 시계, 카운터           |
| LED Matrix   | 여러 LED를 배열한 표시 장치      | 문자, 간단한 그래픽 표시    |
| LCD/OLED     | 문자와 그래픽 표시             | 센서값 출력, 사용자 인터페이스 |

예를 들어 센서등은 PIR 센서가 사람을 감지하면 LED 조명을 켜는 구조로 동작한다.

---

## 9. 모터 기반 액추에이터

모터는 전기 에너지를 회전 운동이나 직선 운동으로 바꾸는 대표적인 액추에이터이다.

| 모터 종류     | 특징              | 사용 예           |
| --------- | --------------- | -------------- |
| DC 모터     | 연속 회전, 속도 제어 가능 | 바퀴, 팬, 펌프      |
| 서보 모터     | 특정 각도로 위치 제어 가능 | 로봇팔, 창문 개폐 장치  |
| 스텝 모터     | 일정 각도씩 정밀하게 회전  | 3D 프린터, CNC 장비 |
| BLDC 모터   | 효율이 높고 수명이 길다   | 드론, 전기차, 팬     |
| 리니어 액추에이터 | 직선 운동을 만든다      | 자동 문, 높이 조절 장치 |

모터 기반 액추에이터는 힘과 움직임을 만들어내기 때문에 로봇, 자동문, 환풍기, 펌프, 드론 등에 많이 사용된다.

---

## 10. 그 외 액추에이터

| 액추에이터 | 출력 형태        | 사용 예           |
| ----- | ------------ | -------------- |
| 부저    | 소리           | 알람, 경고음        |
| 스피커   | 음성, 음악       | 안내 방송, 음성 출력   |
| 릴레이   | 전기 회로 ON/OFF | 조명, 펌프, 히터 제어  |
| 솔레노이드 | 직선 운동        | 도어락, 잠금장치      |
| 전자석   | 자기력          | 물체 고정, 잠금장치    |
| 히터    | 열            | 온도 제어 장치       |
| 펌프    | 물 또는 공기 이동   | 자동 급수, 냉각 시스템  |
| 밸브    | 유체 흐름 제어     | 관수 시스템, 공압 시스템 |

릴레이는 실제로 빛이나 움직임을 직접 만들지는 않지만, 고전압 장비나 큰 부하를 켜고 끄는 출력 제어 장치로 볼 수 있다.

---

## 11. 센서와 액추에이터의 관계

센서와 액추에이터는 임베디드 시스템에서 입력과 출력의 관계를 만든다.

```text
센서 → 마이크로컨트롤러 → 액추에이터
입력       판단/처리          출력
```

예를 들어 온도가 높아지면 쿨링팬을 켜는 시스템은 다음과 같이 구성된다.

```text
온도 센서 → 아두이노 → DC 팬
온도 측정    조건 판단    냉각 동작
```

즉, 센서는 상황을 감지하고, 액추에이터는 그 상황에 대한 실제 동작을 수행한다.

---

## 12. 센서와 액추에이터 조합으로 만들 수 있는 임베디드 장치

| 센서       | 액추에이터   | 만들 수 있는 장치  | 동작 방식            |
| -------- | ------- | ----------- | ---------------- |
| 빗물 센서    | 서보 모터   | 비 오면 닫히는 창문 | 빗물이 감지되면 창문을 닫음  |
| 토양 수분 센서 | 워터 펌프   | 자동 급수 화분    | 수분이 부족하면 물을 공급   |
| 온도 센서    | 쿨링팬     | 자동 냉각 장치    | 온도가 올라가면 팬 동작    |
| 조도 센서    | LED 조명  | 자동 가로등      | 어두워지면 조명 ON      |
| PIR 센서   | LED 조명  | 센서등         | 사람이 감지되면 조명 ON   |
| 초음파 센서   | 부저      | 후방 감지기      | 물체가 가까우면 경고음 발생  |
| 가스 센서    | 부저/환풍기  | 가스 누출 경보 장치 | 가스가 감지되면 경고 및 환기 |
| 문 열림 센서  | 솔레노이드 락 | 스마트 도어락     | 인증 후 잠금 해제       |
| 온습도 센서   | 히터/가습기  | 스마트 온습도 조절기 | 온습도 조건에 따라 장치 동작 |
| 적외선 센서   | 모터      | 자동 손세정기     | 손이 감지되면 펌프 동작    |

---

## 13. 센서와 액추에이터가 함께 붙어 하나의 장치를 이루는 경우

일부 장비는 센서와 액추에이터가 하나의 제품 안에 함께 들어 있다.
이 경우 사용자는 단순히 제품을 사용하지만, 내부에서는 센서 입력과 액추에이터 출력이 계속 반복된다.

| 장비      | 센서                     | 액추에이터                |
| ------- | ---------------------- | -------------------- |
| 로봇청소기   | 거리 센서, 충돌 센서, 바닥 센서    | 바퀴 모터, 흡입 모터, 브러시 모터 |
| 스마트 에어컨 | 온도 센서, 습도 센서, 인체 감지 센서 | 압축기, 팬 모터, 루버 모터     |
| 자동문     | 적외선 센서, 모션 센서          | 도어 모터                |
| 자동 수도꼭지 | 적외선 센서                 | 솔레노이드 밸브             |
| 스마트 도어락 | 키패드, RFID, 지문 센서       | 솔레노이드 락, 부저, LED     |
| 자동차     | 속도 센서, 온도 센서, 거리 센서    | 엔진 제어기, 모터, 브레이크, 조명 |
| 세탁기     | 수위 센서, 무게 센서, 문 잠금 센서  | 모터, 급수 밸브, 배수 펌프     |
| 냉장고     | 온도 센서, 문 열림 센서         | 압축기, 팬, 내부 조명        |

## 14. 실제 생활 속 임베디드 장비 분석

## 14.1 해가 지면 자동으로 켜지는 가로등

| 구분    | 내용                         |
| ----- | -------------------------- |
| 센서    | 조도 센서                      |
| 액추에이터 | LED 조명 또는 가로등 램프           |
| 동작    | 주변 밝기가 기준값보다 낮아지면 조명이 켜진다. |

```text
조도 센서 → 제어기 → 가로등 ON/OFF
```

가로등은 조도 센서를 통해 밤이 되었는지 판단하고, 필요할 때 조명을 켠다.

---

## 14.2 사람이 다가가면 켜지는 센서등

| 구분    | 내용                        |
| ----- | ------------------------- |
| 센서    | PIR 인체 감지 센서              |
| 액추에이터 | LED 조명                    |
| 동작    | 사람이 감지되면 일정 시간 동안 조명을 켠다. |

```text
PIR 센서 → 제어기 → LED 조명 ON
```

센서등은 사람이 없는 시간에는 꺼져 있다가, 사람이 접근하면 자동으로 켜져 전력을 절약한다.

---

## 14.3 자동 급수 화분

| 구분    | 내용                        |
| ----- | ------------------------- |
| 센서    | 토양 수분 센서                  |
| 액추에이터 | 워터 펌프 또는 밸브               |
| 동작    | 흙의 수분이 부족하면 자동으로 물을 공급한다. |

```text
토양 수분 센서 → 아두이노 → 펌프 ON
```

이 장치는 식물 관리에 필요한 물 공급을 자동화할 수 있다.

## 14.4 비가 오면 닫히는 창문

| 구분    | 내용                     |
| ----- | ---------------------- |
| 센서    | 빗물 센서                  |
| 액추에이터 | 서보 모터 또는 리니어 액추에이터     |
| 동작    | 빗물이 감지되면 창문을 자동으로 닫는다. |

```text
빗물 센서 → 제어기 → 창문 모터 동작
```

창문을 깜빡하고 열어둔 상태에서 비가 올 때 실내로 비가 들어오는 것을 막을 수 있다.

---

## 14.5 온도가 올라가면 동작하는 쿨링팬

| 구분    | 내용                     |
| ----- | ---------------------- |
| 센서    | 온도 센서                  |
| 액추에이터 | DC 팬                   |
| 동작    | 온도가 기준값 이상이면 팬을 작동시킨다. |

```text
온도 센서 → 제어기 → 팬 모터 ON
```

컴퓨터, 전원장치, 배터리 시스템, 공조 장치 등에 사용할 수 있다.

---

## 14.6 로봇청소기

| 구분    | 내용                        |
| ----- | ------------------------- |
| 센서    | 거리 센서, 충돌 센서, 바닥 센서       |
| 액추에이터 | 바퀴 모터, 흡입 모터, 브러시 모터      |
| 동작    | 장애물을 감지하며 이동하고, 먼지를 흡입한다. |

로봇청소기는 센서와 액추에이터가 결합된 대표적인 임베디드 시스템이다.

## 15. 센서 측정값 오류 처리 방법

센서값은 항상 정확하지 않다.
노이즈, 배선 불량, 주변 환경, 센서 응답 시간 때문에 값이 순간적으로 튀거나 잘못 측정될 수 있다.

---

## 15.1 정상 범위 검사

센서값이 물리적으로 말이 안 되는 범위라면 오류로 판단한다.

| 센서       | 정상 범위 예시  |
| -------- | --------- |
| 조도 센서    | 0~1023    |
| 온도 센서    | -20℃~80℃  |
| 초음파 센서   | 2cm~400cm |
| 토양 수분 센서 | 0~1023    |
| 습도 센서    | 0~100%    |

예를 들어 습도가 120%로 측정되면 오류로 판단할 수 있다.

---

## 15.2 튀는 값 제거

이전 값과 현재 값의 차이가 너무 크면 순간 오류일 가능성이 있다.

```text
이전값 = 300
현재값 = 950
차이 = 650
→ 갑자기 너무 크게 변했으므로 오류 가능성 있음
```

이 경우 바로 제어에 사용하지 않고 한 번 더 측정하거나 이전 값을 유지한다.

---

## 15.3 오류 횟수 버퍼링

한 번의 오류로 바로 고장이라고 판단하지 않고, 오류가 여러 번 연속으로 발생했을 때만 오류 상태로 처리한다.

```text
오류 1회 발생 → 무시
오류 2회 발생 → 재측정
오류 3회 이상 연속 발생 → 센서 오류로 판단
```

예시 로직은 다음과 같다.

```cpp
if (sensorValue < minValue || sensorValue > maxValue) {
    errorCount++;
} else {
    errorCount = 0;
}

if (errorCount >= 3) {
    sensorError = true;
}
```

---

## 15.4 이동 평균 필터

여러 번 측정한 값을 평균 내면 순간적인 노이즈를 줄일 수 있다.

```cpp
int readAverage(int pin) {
    long sum = 0;

    for (int i = 0; i < 10; i++) {
        sum += analogRead(pin);
        delay(5);
    }

    return sum / 10;
}
```

이동 평균은 센서값을 안정적으로 만들지만, 너무 많이 평균을 내면 반응 속도가 느려질 수 있다.

---

## 15.5 히스테리시스 적용

센서값이 기준값 근처에서 흔들리면 액추에이터가 계속 켜졌다 꺼질 수 있다.
이 문제를 줄이기 위해 켜지는 기준과 꺼지는 기준을 다르게 설정한다.

예를 들어 조도 센서 기반 조명 제어는 다음과 같이 설정할 수 있다.

```text
조명 ON 기준  : 조도값 300 미만
조명 OFF 기준 : 조도값 400 초과
```

이렇게 하면 조도값이 300 근처에서 조금 흔들려도 조명이 불필요하게 반복 동작하지 않는다.

---

## 16. 액추에이터와 동작 타이밍 맞추기

센서값을 읽은 직후 바로 액추에이터를 동작시키면 오동작이 발생할 수 있다.
센서 응답 시간, 액추에이터 초기화 시간, 기계적 동작 시간을 고려해야 한다.

| 고려사항        | 설명                           |
| ----------- | ---------------------------- |
| 센서 안정화 시간   | 전원을 켠 직후 센서값이 안정될 때까지 대기     |
| 측정 주기       | 센서를 너무 빠르게 읽으면 노이즈가 증가할 수 있음 |
| 액추에이터 반응 시간 | 모터, 릴레이, 펌프는 실제 동작까지 시간이 걸림  |
| 기계적 지연      | 창문, 문, 리니어 액추에이터는 이동 시간이 필요  |
| 안전 대기 시간    | 동작 후 다음 명령까지 일정 시간 대기        |

예를 들어 비가 오면 창문이 닫히는 장치에서는 빗물 센서가 한 번 감지되었다고 바로 창문을 닫기보다, 일정 시간 동안 계속 비가 감지되는지 확인한 뒤 모터를 동작시키는 것이 안정적이다.

---

## 17. 여러 개의 센서와 액추에이터 제어

여러 센서와 액추에이터를 동시에 사용하는 경우에는 제어 우선순위를 정해야 한다.

예를 들어 스마트 온실 시스템에서는 다음과 같은 입출력이 있을 수 있다.

| 센서       | 액추에이터   |
| -------- | ------- |
| 온도 센서    | 팬, 히터   |
| 습도 센서    | 가습기     |
| 토양 수분 센서 | 워터 펌프   |
| 조도 센서    | LED 식물등 |
| 빗물 센서    | 창문 모터   |

이때 모든 장치가 동시에 동작하면 전력 소비가 커지고 제어가 불안정할 수 있다.
따라서 다음과 같이 우선순위를 정할 수 있다.

| 우선순위 | 제어 항목    | 이유                  |
| ---- | -------- | ------------------- |
| 1    | 안전 관련 동작 | 과열, 가스, 침수 등은 즉시 대응 |
| 2    | 모터/펌프 제어 | 실제 환경을 크게 변화시키는 장치  |
| 3    | 조명 제어    | 상대적으로 위험도가 낮음       |
| 4    | 표시 장치    | LCD, LED 상태 표시      |

여러 액추에이터를 동기화해야 하는 경우에는 피드백 센서를 사용하여 위치나 속도를 확인해야 한다.
예를 들어 두 개의 리니어 액추에이터를 동시에 움직일 때는 각 액추에이터의 위치 피드백을 읽고, 속도를 조절하여 같은 위치로 이동하도록 제어해야 한다.

---

## 18. 센서와 액추에이터 통합 제어 흐름

센서와 액추에이터를 결합한 임베디드 시스템은 일반적으로 다음 순서로 동작한다.

```text
1. 센서 초기화
2. 액추에이터 초기화
3. 센서 데이터 읽기
4. 센서값 오류 검사
5. 센서값 필터링
6. 조건 판단
7. 액추에이터 제어
8. 상태 저장 또는 표시
9. 반복 실행
```

기본 구조는 다음과 같다.

```cpp
void loop() {
    sensorValue = readSensor();

    if (isError(sensorValue)) {
        handleError();
        return;
    }

    filteredValue = filter(sensorValue);

    if (filteredValue > threshold) {
        actuatorOn();
    } else {
        actuatorOff();
    }

    delay(100);
}
```

---

## 19. 센서와 액추에이터 조합 예시 코드

아래 코드는 조도 센서값에 따라 LED를 제어하는 간단한 예시이다.

```cpp
const int lightSensorPin = A0;
const int ledPin = 9;

int threshold = 300;

void setup() {
    Serial.begin(9600);
    pinMode(ledPin, OUTPUT);
}

void loop() {
    int lightValue = analogRead(lightSensorPin);

    Serial.print("Light: ");
    Serial.println(lightValue);

    if (lightValue < threshold) {
        analogWrite(ledPin, 255);   // 어두우면 LED 밝게
    } else {
        analogWrite(ledPin, 0);     // 밝으면 LED 끄기
    }

    delay(200);
}
```

## 20. 센서와 액추에이터 비교 정리

| 구분             | 센서                  | 액추에이터           |
| -------------- | ------------------- | --------------- |
| 역할             | 입력장치                | 출력장치            |
| 기능             | 환경이나 상태를 감지         | 실제 동작이나 출력을 수행  |
| 신호 흐름          | 외부 환경 → 전기 신호       | 전기 신호 → 물리적 출력  |
| 예시             | 온도 센서, 조도 센서, 거리 센서 | LED, 모터, 부저, 펌프 |
| 임베디드 시스템에서의 역할 | 판단 기준 제공            | 판단 결과 실행        |

센서는 외부 정보를 입력받고, 액추에이터는 시스템의 판단 결과를 실제 동작으로 표현한다.

## 21. 결론

센서는 임베디드 시스템의 입력장치로서 온도, 습도, 빛, 거리, 움직임, 가스, 수분 등 다양한 물리 값을 측정한다.
센서는 아날로그 방식으로 값을 출력할 수도 있고, 처음부터 디지털 데이터로 값을 전달할 수도 있다.

액추에이터는 임베디드 시스템의 출력장치로서 LED, 모터, 부저, 릴레이, 펌프, 밸브, 히터처럼 전기 신호를 실제 출력으로 바꾸는 장치이다.
즉, 센서가 “상황을 감지하는 장치”라면 액추에이터는 “동작을 수행하는 장치”이다.

실제 임베디드 장비는 대부분 센서와 액추에이터가 함께 사용된다.
예를 들어 자동 가로등은 조도 센서와 LED 조명, 센서등은 PIR 센서와 조명, 자동 급수 화분은 토양 수분 센서와 펌프로 구성된다.

센서값은 노이즈와 오류가 발생할 수 있으므로 정상 범위 검사, 이동 평균 필터, 오류 횟수 버퍼링, 히스테리시스 등을 적용해야 한다.
또한 액추에이터는 동작 시간이 필요하므로 센서 응답 시간과 액추에이터 타이밍을 함께 고려해야 안정적인 임베디드 시스템을 만들 수 있다.
