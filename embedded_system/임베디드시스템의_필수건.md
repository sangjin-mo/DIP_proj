# 임베디드 시스템의 필수 조건

## 1. 수행 목표

본 문서는 아두이노 기반 임베디드 시스템을 구성하기 위해 필요한 하드웨어, 소프트웨어, 전력 설계, 데이터 전송 방법을 정리하기 위해 작성하였다.
또한 시골마을의 공원에 설치할 수 있는 조명 제어 시스템을 구상하고, 실제 환경에 필요한 부품과 동작 방식을 정리한다.

---

## 2. 최적의 하드웨어

### 2.1 아두이노에 사용되는 마이크로컨트롤러 종류

아두이노는 하나의 보드만 의미하는 것이 아니라, 다양한 마이크로컨트롤러를 사용하는 개발 보드 계열이다.
초기 아두이노는 주로 8-bit AVR 계열을 사용했지만, 최근에는 ARM Cortex-M 계열의 32-bit 마이크로컨트롤러도 많이 사용된다.

| 아두이노 보드               |    마이크로컨트롤러 |   비트 수 |     클럭 | 특징                               |
| --------------------- | ----------: | -----: | -----: | -------------------------------- |
| Arduino UNO R3        |  ATmega328P |  8-bit | 16 MHz | 가장 기본적인 아두이노 보드, 입문용으로 적합        |
| Arduino Mega 2560     |  ATmega2560 |  8-bit | 16 MHz | I/O 핀이 많아 여러 센서와 모듈 연결에 적합       |
| Arduino Leonardo      |  ATmega32U4 |  8-bit | 16 MHz | USB 기능이 내장되어 키보드/마우스 장치 구현 가능    |
| Arduino Due           | AT91SAM3X8E | 32-bit | 84 MHz | ARM Cortex-M3 기반, 연산 성능이 높음      |
| Arduino MKR WiFi 1010 |      SAMD21 | 32-bit | 48 MHz | Wi-Fi/Bluetooth 기반 IoT 장비 제작에 적합 |
| Arduino Nano 33 BLE   |    nRF52840 | 32-bit | 64 MHz | BLE 통신과 저전력 장비 제작에 적합            |

---

### 2.2 성능 비교

| 구분     | 8-bit AVR 계열                       | 32-bit ARM 계열             |
| ------ | ---------------------------------- | ------------------------- |
| 대표 MCU | ATmega328P, ATmega2560, ATmega32U4 | SAMD21, SAM3X8E, nRF52840 |
| 연산 성능  | 낮음                                 | 높음                        |
| 전력 소비  | 비교적 낮음                             | 보드와 기능에 따라 다름             |
| 메모리 용량 | 작음                                 | 큼                         |
| 사용 난이도 | 쉬움                                 | 상대적으로 복잡                  |
| 적합한 용도 | 단순 센서 제어, LED, 모터 제어               | IoT, 통신, 복잡한 센서 처리, 고속 제어 |

아두이노 UNO는 간단한 센서 제어와 릴레이 제어에는 충분하다.
그러나 데이터 처리량이 많거나 무선 통신, 여러 센서, 고속 제어가 필요한 경우에는 Mega, MKR, Nano 33 BLE, Due 같은 보드가 더 적합하다.

## 3. 220V 교류 전원 장비의 전력 제어 방법

아두이노는 5V 또는 3.3V 수준의 낮은 전압으로 동작한다.
따라서 220V 교류 전원을 사용하는 조명, 히터, 펌프, 선풍기 같은 장비를 아두이노 핀에 직접 연결하면 안 된다.

220V 장비를 제어하려면 아두이노와 고전압 장비 사이에 전력 제어 모듈을 넣어야 한다.

---

### 3.1 릴레이 모듈 사용

릴레이는 작은 제어 신호로 큰 전력을 켜고 끌 수 있는 스위치 역할을 한다.

```text
아두이노 출력 핀
→ 릴레이 모듈 입력
→ 릴레이 접점 ON/OFF
→ 220V 조명 전원 제어
```

| 항목     | 설명                      |
| ------ | ----------------------- |
| 장점     | 구조가 단순하고 ON/OFF 제어가 쉬움  |
| 단점     | 빠른 스위칭에는 부적합, 접점 수명 존재  |
| 적합한 장비 | 조명, 펌프, 팬, 단순 ON/OFF 장비 |

---

### 3.2 SSR 사용

SSR(Solid State Relay)은 반도체 방식의 릴레이이다.
기계식 접점이 없기 때문에 소음이 적고 반복 동작에 유리하다.

| 항목     | 설명                   |
| ------ | -------------------- |
| 장점     | 무소음, 빠른 스위칭, 긴 수명    |
| 단점     | 발열이 있을 수 있어 방열판 필요   |
| 적합한 장비 | 조명, 히터, 반복 ON/OFF 장비 |

---

### 3.3 트라이악 제어

트라이악(TRIAC)은 교류 전력을 위상 제어할 수 있는 소자이다.
단순히 켜고 끄는 것이 아니라 조명의 밝기 조절이나 히터 출력 조절에 사용할 수 있다.

```text
아두이노
→ 옵토커플러
→ 트라이악
→ 220V AC 부하 제어
```

| 항목     | 설명                    |
| ------ | --------------------- |
| 장점     | 밝기 조절, 출력 조절 가능       |
| 단점     | 회로 설계가 복잡하고 안전 설계가 중요 |
| 적합한 장비 | 조광등, 히터 출력 제어         |

---

### 3.4 220V 제어 시 주의사항

220V 교류 전원은 감전과 화재 위험이 있으므로 직접 배선 실습 시 주의해야 한다.

| 주의사항  | 내용                                 |
| ----- | ---------------------------------- |
| 절연    | 아두이노 저전압 회로와 220V 고전압 회로를 분리해야 한다. |
| 정격 확인 | 릴레이나 SSR의 허용 전압과 전류를 확인해야 한다.      |
| 퓨즈 사용 | 과전류 발생 시 회로를 보호하기 위해 퓨즈를 사용한다.     |
| 방수/방진 | 실외 설치 시 방수 케이스가 필요하다.              |
| 접지    | 금속 외함을 사용하는 경우 접지 처리가 필요하다.        |

## 4. 탑재되는 소프트웨어

### 4.1 아두이노 펌웨어의 특징

아두이노에서 동작하는 프로그램은 일반적으로 펌웨어라고 볼 수 있다.
펌웨어는 마이크로컨트롤러 내부의 플래시 메모리에 저장되어 전원이 켜질 때 실행된다.

아두이노 프로그램은 기본적으로 다음 구조를 가진다.

```cpp
void setup() {
    // 처음 한 번만 실행되는 초기화 코드
}

void loop() {
    // 전원이 켜져 있는 동안 반복 실행되는 코드
}
```

| 항목             | 설명                      |
| -------------- | ----------------------- |
| setup()        | 핀 모드, 통신 속도, 초기값 설정     |
| loop()         | 센서 읽기, 판단, 출력 제어를 반복 수행 |
| delay()        | 일정 시간 대기                |
| digitalWrite() | 디지털 출력 제어               |
| analogWrite()  | PWM 출력 제어               |
| Serial         | UART/USB 시리얼 통신         |

---

### 4.2 부트로더의 역할

부트로더는 아두이노에 프로그램을 쉽게 업로드할 수 있도록 도와주는 작은 프로그램이다.
PC에서 Arduino IDE로 작성한 코드를 업로드하면, 부트로더가 새 펌웨어를 마이크로컨트롤러의 플래시 메모리에 기록한다.

```text
PC의 Arduino IDE
→ USB 연결
→ 부트로더 실행
→ 컴파일된 바이너리 업로드
→ 플래시 메모리에 저장
→ 펌웨어 실행
```

부트로더가 손상되면 일반적인 USB 업로드가 되지 않을 수 있으며, 이 경우 ISP 장비를 이용하여 부트로더를 다시 기록해야 한다.

---

### 4.3 펌웨어, RTOS, 임베디드 리눅스 비교

| 구분      | 아두이노 펌웨어          | RTOS            | 임베디드 리눅스                       |
| ------- | ----------------- | --------------- | ------------------------------ |
| 구조      | 단일 반복 루프 중심       | 여러 태스크를 스케줄링    | 일반 OS처럼 프로세스 관리                |
| 실시간성    | 단순 제어 가능          | 실시간 제어에 적합      | 기본 Linux는 hard real-time에는 부적합 |
| 복잡도     | 낮음                | 중간              | 높음                             |
| 메모리 요구량 | 작음                | 중간              | 큼                              |
| 부팅 시간   | 빠름                | 빠름              | 상대적으로 느림                       |
| 예시      | Arduino UNO 센서 제어 | FreeRTOS 기반 제어기 | Raspberry Pi, 산업용 Linux 보드     |
| 적합한 용도  | 단순 센서/모터/릴레이 제어   | 로봇, 자동차, 산업 제어  | 네트워크, GUI, 복잡한 응용 프로그램         |

아두이노 펌웨어는 구조가 단순하고 배우기 쉽다.
반면 여러 작업을 동시에 정밀하게 관리해야 하는 경우에는 RTOS가 적합하고, 네트워크·파일 시스템·카메라·GUI 같은 복잡한 기능이 필요한 경우에는 임베디드 리눅스가 적합하다.

## 5. 바이너리 호환성과 컴파일의 필요성

아두이노 보드마다 사용하는 마이크로컨트롤러의 종류가 다르다.
따라서 같은 C/C++ 코드라도 보드에 따라 컴파일 결과가 달라진다.

예를 들어 ATmega328P에서 동작하도록 컴파일된 바이너리는 ARM Cortex-M 계열 보드에서 그대로 실행될 수 없다.

### 이유

| 이유         | 설명                              |
| ---------- | ------------------------------- |
| CPU 구조 차이  | AVR과 ARM은 명령어 구조가 다르다.          |
| 레지스터 구조 차이 | 내부 레지스터와 주변장치 주소가 다르다.          |
| 메모리 구조 차이  | 플래시, SRAM, EEPROM 구성 방식이 다르다.   |
| 주변장치 차이    | 타이머, UART, ADC, PWM 제어 방식이 다르다. |

따라서 보드에 맞는 컴파일 과정이 반드시 필요하다.

```text
소스 코드
→ 보드 선택
→ 해당 MCU용 컴파일러 사용
→ 보드에 맞는 바이너리 생성
→ 펌웨어 업로드
```

## 6. 저전력 시스템 및 경량화

### 6.1 아두이노 전력 소비량 추정

아두이노의 소비 전력은 보드 종류, 동작 전압, 센서 및 모듈 연결 상태에 따라 달라진다.
여기서는 아두이노 UNO가 약 5V, 50mA를 소비한다고 가정하여 계산한다.

### 계산식

```text
소비 전력 P = 전압 V × 전류 I
```

```text
P = 5V × 0.05A = 0.25W
```

즉, 아두이노 UNO 단독 동작 시 약 0.25W를 소비한다고 가정할 수 있다.

---

### 6.2 배터리 동작 시간 계산

5V 2000mAh 보조배터리를 사용한다고 가정한다.

```text
배터리 에너지 = 전압 × 용량
```

```text
배터리 에너지 = 5V × 2Ah = 10Wh
```

아두이노 소비 전력이 0.25W라면 이론상 동작 시간은 다음과 같다.

```text
동작 시간 = 배터리 에너지 / 소비 전력
```

```text
동작 시간 = 10Wh / 0.25W = 40시간
```

하지만 실제로는 전압 변환 손실, 센서와 통신 모듈의 소비 전력, 배터리 효율 저하가 있으므로 실제 동작 시간은 약 25~30시간 정도로 예상할 수 있다.

---

### 6.3 모듈 추가 시 소비 전력 변화

| 구성             |       예상 소비 전류 | 설명            |
| -------------- | -------------: | ------------- |
| Arduino UNO 단독 |         약 50mA | 기본 보드 동작      |
| 조도 센서 추가       |         약 55mA | 센서 전류 소모는 작음  |
| 릴레이 모듈 추가      |     약 70~100mA | 릴레이 코일 전류 필요  |
| Wi-Fi 모듈 추가    | 약 150~300mA 이상 | 무선 통신 시 전류 증가 |
| 여러 모듈 동시 사용    |       300mA 이상 | 배터리 용량 증가 필요  |

따라서 배터리 기반 시스템을 만들 때는 센서보다 통신 모듈과 릴레이, 모터의 소비 전류를 더 중요하게 고려해야 한다.

---

### 6.4 초경량 임베디드 시스템 구성

초경량 임베디드 시스템을 만들기 위해서는 Arduino UNO보다 작은 보드를 사용하는 것이 유리하다.
예를 들어 Arduino Pro Mini, Arduino Nano, Seeeduino XIAO 같은 소형 보드를 사용할 수 있다.

| 부품              |    예상 무게 |
| --------------- | -------: |
| 소형 아두이노 보드      |   약 2~7g |
| 조도 센서           |   약 1~2g |
| 소형 배터리          | 약 10~20g |
| BLE 모듈 또는 무선 모듈 |   약 2~5g |
| 케이스 및 배선        |  약 5~10g |

간단한 센서 데이터 수집 장비라면 약 20~40g 수준의 초경량 임베디드 시스템을 구성할 수 있다.
다만 220V 릴레이나 큰 배터리, 방수 케이스를 포함하면 무게는 더 증가한다.

## 7. 데이터 수집, 저장 및 전송 방법

아두이노는 센서 데이터를 수집하고, 저장하거나 외부 장치로 전송할 수 있다.

### 7.1 데이터 수집

| 센서     | 수집 데이터    |
| ------ | --------- |
| 조도 센서  | 밝기        |
| 온습도 센서 | 온도, 습도    |
| 초음파 센서 | 거리        |
| PIR 센서 | 사람 움직임 감지 |
| 전류 센서  | 부하 전류     |
| 전압 센서  | 배터리 전압    |

---

### 7.2 데이터 저장 방법

| 저장 방법          | 설명                      |
| -------------- | ----------------------- |
| Serial Monitor | PC에 연결해 실시간으로 데이터 확인    |
| SD 카드 모듈       | CSV 파일 형태로 장기간 데이터 저장   |
| EEPROM         | 소량의 설정값 저장              |
| 외부 Flash 메모리   | 비교적 많은 데이터 저장           |
| 클라우드 저장        | Wi-Fi 모듈을 통해 서버에 데이터 전송 |

SD 카드 모듈을 사용하면 다음과 같은 형태로 데이터를 저장할 수 있다.

```csv
time, light, motion, lamp_state
08:00, 320, 0, OFF
18:30, 120, 1, ON
23:10, 80, 0, DIM
```

---

### 7.3 데이터 전송 방법

| 전송 방식      | 특징                | 사용 예시           |
| ---------- | ----------------- | --------------- |
| USB Serial | PC와 직접 연결         | 디버깅, 실험 데이터 확인  |
| UART       | 다른 MCU나 통신 모듈과 연결 | 센서 보드, 블루투스 모듈  |
| Bluetooth  | 근거리 무선 통신         | 스마트폰 앱 연동       |
| Wi-Fi      | 인터넷 연결 가능         | 서버 전송, IoT 장비   |
| LoRa       | 장거리 저전력 통신        | 농촌, 공원, 환경 모니터링 |
| Ethernet   | 안정적인 유선 통신        | 고정형 설비, 산업 현장   |
| CAN        | 차량·산업용 제어기 통신     | 모터 제어기, 차량 ECU  |

시골마을 공원 조명 제어 시스템에서는 장거리 저전력 통신이 필요할 수 있으므로 LoRa 또는 Wi-Fi를 고려할 수 있다.
단, 공원에 인터넷 공유기가 없거나 거리가 멀다면 LoRa 방식이 더 적합할 수 있다.

## 8. 시골마을 공원 조명 제어 시스템 구상

### 8.1 시스템 목적

시골마을 공원에 설치되는 조명을 자동으로 제어하여 야간 보행 안전을 높이고, 불필요한 전력 소비를 줄이는 시스템을 구상한다.

기본 목표는 다음과 같다.

1. 주변이 어두워지면 자동으로 조명을 켠다.
2. 사람이 감지되면 밝기를 높인다.
3. 사람이 없을 때는 밝기를 낮추거나 일부 조명만 켠다.
4. 고장 또는 이상 상태를 기록한다.
5. 필요 시 원격으로 조명 상태를 확인한다.

---

### 8.2 시스템 구성

| 구성 요소      | 역할                |
| ---------- | ----------------- |
| Arduino 보드 | 센서 입력 처리 및 조명 제어  |
| 조도 센서      | 주변 밝기 측정          |
| PIR 센서     | 사람 움직임 감지         |
| 릴레이 또는 SSR | 220V 조명 ON/OFF 제어 |
| RTC 모듈     | 시간 기반 제어          |
| SD 카드 모듈   | 동작 로그 저장          |
| 통신 모듈      | 관리자에게 상태 전송       |
| 전원 장치      | 아두이노 및 센서 전원 공급   |
| 방수 케이스     | 실외 환경 보호          |

---

### 8.3 동작 흐름

```text
조도 센서로 주변 밝기 측정
→ 어두운지 판단
→ PIR 센서로 사람 움직임 감지
→ 조명 ON/OFF 또는 밝기 제어
→ 동작 상태를 SD 카드에 저장
→ 필요 시 관리자에게 상태 전송
```

---

### 8.4 제어 조건

| 조건              | 동작               |
| --------------- | ---------------- |
| 낮이고 밝음          | 조명 OFF           |
| 밤이고 사람이 없음      | 조명 약하게 ON 또는 OFF |
| 밤이고 사람이 감지됨     | 조명 ON            |
| 일정 시간 동안 움직임 없음 | 조명 밝기 감소 또는 OFF  |
| 센서 오류 발생        | 기본 안전 모드로 조명 ON  |
| 통신 장애 발생        | 로컬 제어 모드로 계속 동작  |

---

### 8.5 예시 제어 알고리즘

```cpp
if (light_value < threshold_dark) {
    if (motion_detected == true) {
        lamp_on();
        save_log("NIGHT_MOTION_LAMP_ON");
    } else {
        lamp_dim_or_off();
        save_log("NIGHT_NO_MOTION_DIM");
    }
} else {
    lamp_off();
    save_log("DAY_LAMP_OFF");
}
```

---

### 8.6 데이터 로그 예시

```csv
time, light_value, motion, lamp_state
18:10, 180, 0, OFF
18:40, 90, 1, ON
19:20, 70, 0, DIM
23:30, 50, 0, OFF
```

이 로그를 통해 조명이 언제 켜졌는지, 사람이 감지되었는지, 센서값이 정상인지 확인할 수 있다.

## 11. 최적의 하드웨어 선정

공원 조명 제어 시스템은 복잡한 영상 처리나 고속 연산이 필요하지 않다.
따라서 기본 제어용으로는 Arduino UNO 또는 Nano급 보드도 사용할 수 있다.

그러나 통신 모듈, 여러 센서, 로그 저장 기능을 함께 사용한다면 다음과 같은 구성이 적합하다.

| 목적              | 추천 보드                    |
| --------------- | ------------------------ |
| 단순 조명 ON/OFF 제어 | Arduino UNO              |
| 소형화 및 저전력       | Arduino Nano 또는 Pro Mini |
| 센서와 출력이 많은 경우   | Arduino Mega 2560        |
| Wi-Fi 기반 IoT 제어 | Arduino MKR WiFi 1010    |
| BLE 기반 저전력 통신   | Arduino Nano 33 BLE      |

본 과제에서는 시골마을 공원 조명 제어 시스템이므로, 유지보수와 구현 난이도를 고려하여 **Arduino UNO + 릴레이 모듈 + 조도 센서 + PIR 센서 + SD 카드 모듈** 구성을 기본안으로 선택한다.

## 13. 정리

아두이노는 사용하는 보드에 따라 8-bit AVR부터 32-bit ARM Cortex-M 계열까지 다양한 마이크로컨트롤러를 사용한다.
보드마다 성능, 메모리, 핀 수, 전력 소비량, 통신 기능이 다르므로 목적에 맞는 하드웨어 선택이 중요하다.

아두이노는 단순한 펌웨어 구조를 가지며, 센서 입력과 출력 제어를 빠르게 구현할 수 있다.
하지만 복잡한 실시간 태스크 관리가 필요한 경우에는 RTOS가 적합하고, 네트워크와 파일 시스템, 고성능 응용 프로그램이 필요한 경우에는 임베디드 리눅스가 적합하다.

시골마을 공원 조명 제어 시스템은 아두이노, 조도 센서, PIR 센서, 릴레이 또는 SSR, 데이터 저장 모듈을 활용하여 구현할 수 있다.
이를 통해 야간 보행 안전을 높이고, 불필요한 전력 소비를 줄이는 임베디드 시스템을 구성할 수 있다.
