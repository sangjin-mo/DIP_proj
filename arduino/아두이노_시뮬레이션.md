# 임베디드 리눅스 연동 아두이노 회로 설계 및 시뮬레이션

## 1. 수행 목표

Fritzing과 Autodesk 계열 도구를 이용하여 다음 두 아두이노 예제를 역설계하고 시뮬레이션하는 방법을 정리한다.

1. GPIO LED를 이용한 S.O.S 점멸 회로
2. DS1307 RTC와 MAX7219 8×8 LED 매트릭스를 이용한 한글시계

최종 장치는 Raspberry Pi와 같은 임베디드 리눅스 보드가 상위 제어를 담당하고, Arduino UNO가 LED와 RTC를 실시간으로 제어하는 구조로 설계한다.

> Arduino UNO 자체는 임베디드 리눅스 장치가 아니라 마이크로컨트롤러 보드이다. 따라서 본 설계에서는 Raspberry Pi를 임베디드 리눅스 장치로 두고 Arduino를 USB Serial 기반 I/O 제어기로 사용한다.

## 2. 도구의 역할과 현재 서비스 상태

| 도구 | 본 과제에서의 역할 | 주의사항 |
| --- | --- | --- |
| Fritzing | 브레드보드 배선도, 회로도, PCB 초안 작성 및 이미지 출력 | Arduino 프로그램 전체를 정밀하게 실행하는 용도는 아님 |
| Fritzing Simulator | LED 전류 등 단순 DC 회로 확인 | 공식 설명상 DC 회로 중심의 제한적인 시뮬레이터 |
| library.io | 부품 패키지, 풋프린트 및 3D 모델 검색·생성 | 회로 동작 시뮬레이터가 아님 |
| Tinkercad Circuits | Arduino 코드 실행, LED 점멸 및 브레드보드 동작 확인 | 예전 circuits.io/123D Circuits 기능을 대신함 |
| Wokwi | Tinkercad에 없는 RTC·MAX7219 부품의 보조 시뮬레이션 | 과제 지정 도구를 보완하는 용도로 사용 |

Fritzing 공식 문서에서는 Breadboard, Schematic, PCB 보기를 제공하며 한 보기에서 수정한 연결이 다른 보기에도 반영된다고 설명한다. Fritzing의 시뮬레이터는 0.9.10 이후 제공되지만 현재는 DC 회로 분석 중심으로 기능이 제한되어 있다.

기존 `circuits.io`의 Arduino 회로 편집·시뮬레이션 기능은 종료되어 Tinkercad Circuits로 이전되었다. 현재 `library.io`는 Autodesk Fusion과 연동하는 부품 라이브러리 및 풋프린트 서비스이다. 따라서 과제의 도구 이름은 다음처럼 해석하는 것이 정확하다.

```text
회로 문서화       → Fritzing
부품/풋프린트 확인 → library.io
Arduino 시뮬레이션 → Tinkercad Circuits
```

## 3. 전체 시스템 설계

### 3.1 시스템 구성

```text
┌──────────────────────────────────────┐
│ Raspberry Pi / Embedded Linux        │
│ - 시스템 시작 및 상태 기록           │
│ - 현재 시각 전송                     │
│ - 시리얼 로그 저장                   │
└─────────────────┬────────────────────┘
                  │ USB Serial
                  │ 5V USB 전원/통신
┌─────────────────▼────────────────────┐
│ Arduino UNO                          │
│ - GPIO LED S.O.S 출력                │
│ - DS1307 시각 읽기                   │
│ - MAX7219 한글시계 좌표 제어          │
└───────┬───────────────────┬──────────┘
        │ D13               │ SPI 방식 + I2C
   LED + 220Ω          MAX7219 + DS1307
```

### 3.2 역할 분담

| 구성요소 | 역할 |
| --- | --- |
| Raspberry Pi | 임베디드 리눅스 실행, 시간 동기화, 로그 기록 및 상위 애플리케이션 실행 |
| Arduino UNO | 일정한 타이밍이 필요한 GPIO와 LED 매트릭스 제어 |
| USB Serial | Linux와 Arduino 사이의 명령 및 상태 전달 |
| DS1307 RTC | Linux 연결이 끊겨도 시각 유지 |
| MAX7219 | 8×8 LED를 적은 수의 Arduino 핀으로 구동 |
| LED와 220Ω 저항 | GPIO 출력 및 S.O.S 패턴 확인 |

상위 Linux 장치가 재부팅되거나 잠시 연결이 끊겨도 Arduino와 RTC가 시계 표시를 유지할 수 있다. 반대로 Linux 장치는 부팅 후 정확한 시간을 Arduino에 전송하여 RTC 오차를 보정할 수 있다.

## 4. 필요한 부품과 비용 고려

| 부품 | 수량 | 주요 사양 및 선택 이유 |
| --- | ---: | --- |
| Raspberry Pi 또는 동급 SBC | 1 | 임베디드 리눅스 실행, USB 포트 필요 |
| Arduino UNO | 1 | Tinkercad와 Fritzing에서 다루기 쉬움 |
| 빨간 LED | 1 | GPIO 동작 확인 |
| 220Ω 저항 | 1 | LED 전류 제한 |
| MAX7219 8×8 LED 모듈 | 1 | 64개 LED를 3개 제어선으로 구동 |
| DS1307 RTC 모듈 | 1 | I2C 시각 읽기 및 백업 전지 사용 |
| CR2032 코인전지 | 1 | RTC 백업 전원 |

초기 검증은 UNO, RTC, 완성형 MAX7219 모듈을 사용하여 부품 수와 제작 비용을 줄인다. 양산 단계에서만 전용 PCB를 검토한다.

## 5. 회로 1: GPIO LED S.O.S

### 5.1 배선

```text
Arduino D13 ── 220Ω ── LED 애노드(+)
                            LED 캐소드(-) ── GND
```

| Arduino UNO | 연결 부품 | 기능 |
| --- | --- | --- |
| D13 | 220Ω 저항 → LED 애노드 | 디지털 출력 |
| GND | LED 캐소드 | 공통 접지 |

내장 LED만 사용할 때에는 외부 LED와 저항을 생략하고 `LED_BUILTIN`을 사용할 수 있다. 외부 LED를 사용할 때에는 극성과 저항을 반드시 확인한다.

### 5.2 전류 계산

빨간 LED의 순방향 전압을 약 2V라고 가정하면 다음과 같다.

```text
I = (5V - 2V) / 220Ω
  ≒ 0.0136A
  ≒ 13.6mA
```

이는 일반적인 표시용 LED에 적절한 수준이다. 소비전력을 더 줄이려면 330Ω을 사용할 수 있다.

```text
I = (5V - 2V) / 330Ω ≒ 9.1mA
```

### 5.3 동작 코드

기존 `아두이노_GPIO_LED_제어.md`의 S.O.S 코드를 사용한다.

```cpp
const int LED_PIN = 13;
const int UNIT = 200;

void signal(int duration) {
  digitalWrite(LED_PIN, HIGH);
  delay(duration);
  digitalWrite(LED_PIN, LOW);
  delay(UNIT);
}

void outputS() {
  signal(UNIT);
  signal(UNIT);
  signal(UNIT);
}

void outputO() {
  signal(UNIT * 3);
  signal(UNIT * 3);
  signal(UNIT * 3);
}

void setup() {
  pinMode(LED_PIN, OUTPUT);
  digitalWrite(LED_PIN, LOW);
}

void loop() {
  outputS();
  delay(UNIT * 2);
  outputO();
  delay(UNIT * 2);
  outputS();
  delay(UNIT * 6);
}
```

### 5.4 동작 원리

`pinMode()`로 D13을 출력으로 만들고 `digitalWrite()`로 5V와 0V 상태를 반복한다. `UNIT`을 200ms로 정하면 점은 200ms, 선은 600ms 동안 LED가 켜진다.

```text
S: 200ms ON × 3
O: 600ms ON × 3
S: 200ms ON × 3
반복 간격: 1,400ms
```

## 6. 회로 2: 8×8 LED 한글시계

### 6.1 MAX7219 연결

| Arduino UNO | MAX7219 | 기능 |
| --- | --- | --- |
| 5V | VCC | 모듈 전원 |
| GND | GND | 공통 접지 |
| D11 | DIN | 직렬 데이터 |
| D13 | CLK | 클럭 |
| D10 | CS/LOAD | 데이터 적용 |

### 6.2 DS1307 연결

| Arduino UNO | DS1307 | 기능 |
| --- | --- | --- |
| 5V | VCC | 모듈 전원 |
| GND | GND | 공통 접지 |
| A4 | SDA | I2C 데이터 |
| A5 | SCL | I2C 클럭 |

모든 장치의 GND를 연결해야 신호 전압의 기준이 같아진다. MAX7219 모듈 가까이에 100µF와 0.1µF 커패시터를 병렬로 배치하면 화면 전환 시 발생하는 전원 흔들림을 줄일 수 있다.

### 6.3 동작 원리

1. DS1307이 현재 시각을 BCD 형태로 보관한다.
2. Arduino가 I2C를 통해 시와 분을 읽는다.
3. 24시간 값을 12시간 값과 오전·오후로 변환한다.
4. 시간에 해당하는 한글 문자 좌표를 결정한다.
5. Arduino가 DIN, CLK, CS를 통해 MAX7219에 8개 행의 점등 데이터를 전송한다.
6. MAX7219가 LED를 빠르게 다중화하여 한글 문자판의 필요한 위치를 켠다.

예를 들어 `03:25`에는 다음 좌표가 켜진다.

```text
오 + 전 + 세 + 시 + 이 + 십 + 오 + 분
→ 오전 세시 이십오분
```

### 6.4 실제 구현 시 보완할 코드

기존 한글시계 문서의 코드는 구조 설명용 축약본이므로 `displayHour()`의 4시부터 9시까지와 `displayMinute()`의 1분부터 59분까지 좌표 변환을 완성해야 한다. 이 부분을 구현하지 않으면 회로가 정상이어도 일부 시간의 표시가 누락된다.

## 7. Fritzing 회로 설계 방법

### 7.1 LED 회로

1. Fritzing을 실행하고 `File → Save As`로 `arduino_sos.fzz`를 저장한다.
2. Breadboard 보기에서 Arduino UNO, 브레드보드, LED, 220Ω 저항을 배치한다.
3. D13에서 저항, LED 애노드 순서로 연결한다.
4. LED 캐소드를 Arduino GND에 연결한다.
5. LED의 `forward voltage`와 저항값을 Inspector에서 확인한다.
6. Schematic 보기로 전환하여 회로 기호와 연결점을 정돈한다.
7. `File → Export → as Image → PNG`로 회로 이미지를 저장한다.

### 7.2 한글시계 회로

1. 새 파일을 `arduino_korean_clock.fzz`로 저장한다.
2. Arduino UNO, MAX7219 모듈, DS1307 모듈, 전원 커패시터를 배치한다.
3. 5V는 빨간색, GND는 검은색, 신호선은 다른 색으로 구분한다.
4. D11-DIN, D13-CLK, D10-CS를 연결한다.
5. A4-SDA와 A5-SCL을 연결한다.
6. Schematic 보기에서 전원망과 신호망 이름을 표시한다.
7. ERC를 대신하여 미연결 핀, 교차선, 공통 GND를 수동 점검한다.
8. 필요하면 PCB 보기에서 부품을 배치하되 전원 배선은 신호선보다 굵게 한다.

Fritzing에 정확한 RTC 또는 MAX7219 모듈이 없으면 `Mystery Part`로 커넥터 이름을 정의하거나 Part Creator로 부품을 추가한다. library.io에서 검색한 풋프린트는 핀 번호와 실제 데이터시트를 대조한 후 사용한다.

### 7.3 Fritzing DC 시뮬레이션

Fritzing의 `Edit → Preferences → Beta Features → Enable Simulator`를 활성화한다.

LED 회로에 5V 전원과 멀티미터를 연결하여 다음을 확인한다.

| 측정 항목 | 예상값 | 판정 기준 |
| --- | ---: | --- |
| LED 직렬 전류 | 약 13.6mA | LED 최대 정격보다 작아야 함 |
| 저항 전압 강하 | 약 3V | LED 순방향 전압에 따라 달라짐 |
| LED 양단 전압 | 약 2V | LED 색상과 모델에 따라 달라짐 |

Fritzing 시뮬레이터는 Arduino 스케치의 전체 시간 동작을 확인하기보다 정적 DC 전류와 잘못된 과전류 상태를 점검하는 용도로 사용한다.

## 8. library.io 사용 방법

1. `https://library.io/`에 접속한다.
2. `DS1307`, `MAX7219`, 사용하려는 커넥터를 검색한다.
3. 패키지의 핀 수, 핀 간격, 본체 크기를 데이터시트와 비교한다.
4. Autodesk Fusion 라이브러리와 동기화하거나 지원되는 형식으로 내보낸다.
5. PCB 제작 전 실물 부품의 치수를 다시 측정한다.

library.io의 패키지는 PCB 풋프린트 제작에 유용하지만 Arduino 코드나 LED 점멸을 실행하지 않는다. 또한 사이트에서도 제조 전에 부품을 검증하도록 안내하므로 검색 결과를 그대로 양산에 사용하면 안 된다.

## 9. Tinkercad Circuits 시뮬레이션

### 9.1 LED S.O.S 시뮬레이션

1. `https://www.tinkercad.com/circuits`에 접속한다.
2. `Create new Circuit`를 선택한다.
3. Arduino UNO, LED, 220Ω 저항, 브레드보드를 배치한다.
4. 5장의 배선표와 동일하게 연결한다.
5. `Code → Text`를 선택하고 S.O.S 코드를 붙여 넣는다.
6. `Start Simulation`을 누른다.
7. LED가 짧게 3회, 길게 3회, 짧게 3회 점멸하는지 확인한다.

### 9.2 확인해야 할 시간

스톱워치 또는 시뮬레이터 시간 기준으로 다음을 확인한다.

| 구간 | 예상 점등 시간 |
| --- | ---: |
| S의 각 점 | 200ms |
| O의 각 선 | 600ms |
| 신호 내부 간격 | 200ms |
| 문자 사이 전체 간격 | 600ms |
| S.O.S 반복 사이 전체 간격 | 1,400ms |

### 9.3 한글시계 시뮬레이션

Tinkercad의 부품 목록에 사용 중인 MAX7219 모듈 또는 DS1307 모듈이 없다면 억지로 다른 부품에 연결하지 않는다. 다음처럼 기능을 나누어 검증한다.

1. Tinkercad에서 Arduino의 시간 변환 알고리즘을 Serial Monitor 출력으로 확인한다.
2. 입력 시간을 `03:25`, `08:48`, `12:00`, `23:59`로 고정하여 한글 좌표 목록을 출력한다.
3. MAX7219와 DS1307의 통합 동작은 Wokwi에서 보조 검증한다.
4. Fritzing에서는 실제 배선과 전원 연결을 문서화한다.
5. 최종 하드웨어에서 문자판 방향과 빛 번짐을 확인한다.

이 방식은 시뮬레이터에 없는 부품을 임의로 대체하여 잘못된 결과를 만드는 것을 방지한다.

## 10. 시뮬레이션 판정표

현재 문서 작성 환경에서는 Fritzing 및 Tinkercad GUI를 직접 조작하여 결과 화면을 저장하지 않았으므로, 아래 표의 `GUI 실행 결과`는 실제 수행 후 작성해야 한다. 코드와 회로를 분석한 예상 판정은 함께 기록한다.

| 시험 항목 | 분석상 예상 결과 | GUI 실행 결과 |
| --- | --- | --- |
| LED D13 HIGH | LED 점등 | 미수행 |
| LED D13 LOW | LED 소등 | 미수행 |
| S.O.S 순서 | 짧게 3회 → 길게 3회 → 짧게 3회 | 미수행 |
| 220Ω LED 전류 | 약 13.6mA | 미수행 |
| RTC I2C 통신 | A4/A5 배선이 맞으면 시각 읽기 | 미수행 |
| 03:25 표시 | 오전 세시 이십오분 | 미수행 |
| 12:00 표시 | 오후 열두시 정각 | 미수행 |
| 전원 재인가 | RTC 전지가 있으면 시각 유지 | 실제 장치 시험 필요 |

시뮬레이션을 수행한 뒤 `미수행`을 `통과` 또는 `실패`로 바꾸고, 실패 시 측정값과 수정 내용을 적는다.

## 11. 예상 문제와 해결 방법

| 문제 | 가능한 원인 | 해결 방법 |
| --- | --- | --- |
| LED가 켜지지 않음 | LED 극성 반대, GND 미연결, 핀 번호 오류 | 애노드·캐소드와 D13 연결 확인 |
| LED가 과열됨 | 저항 누락 또는 너무 작은 저항 | 220~330Ω 직렬 저항 사용 |
| S.O.S 간격이 이상함 | `signal()`의 기본 1단위 지연을 중복 계산 | 문자 간 추가 지연은 2단위, 반복 간 추가 지연은 6단위 사용 |
| RTC를 찾지 못함 | SDA/SCL 반대, 공통 GND 누락 | UNO 기준 A4=SDA, A5=SCL 확인 |
| RTC 시간이 초기화됨 | 코인전지 누락 또는 방전 | CR2032 전압과 삽입 방향 확인 |
| 매트릭스가 뒤집힘 | 모듈 장착 방향과 좌표계 차이 | `setLed()`의 행·열 또는 좌표 테이블 변환 |
| 일부 시간 글자가 누락됨 | 축약 코드의 시·분 좌표 미구현 | 1~12시와 0~59분 좌표 테이블 완성 |
| LED 화면이 깜박임 | 매 반복마다 불필요하게 전체 화면 갱신 | 분이 바뀔 때만 화면 갱신 |
| Arduino가 재부팅됨 | LED 모듈 순간 전류로 5V 강하 | 모듈 근처 100µF+0.1µF, 안정된 5V 전원 사용 |
| Linux에서 포트를 못 찾음 | `/dev/ttyACM*` 권한 또는 장치명 변경 | udev 규칙 또는 `dialout` 그룹 설정, VID/PID 기반 탐색 |

## 12. 효율성과 신뢰성 최적화

### 12.1 전력

- MAX7219 밝기는 필요한 수준까지만 설정한다.
- 여러 LED가 동시에 켜질 때의 최대 전류를 기준으로 5V 전원을 선택한다.
- LED 회로의 저항을 330Ω으로 높이면 시인성을 크게 잃지 않고 전류를 줄일 수 있다.
- Raspberry Pi의 5V 핀에서 큰 LED 부하를 직접 공급하지 않는다.

### 12.2 신호와 배선

- SPI와 I2C 선을 짧게 유지한다.
- 신호선마다 공통 GND 기준을 확보한다.
- 전원선은 신호선보다 굵게 설계한다.
- MAX7219 전원 가까이에 바이패스 커패시터를 둔다.

### 12.3 소프트웨어

- `delay()` 중심 S.O.S 예제는 학습에는 단순하지만 다른 작업을 동시에 처리하기 어렵다.
- 실제 통합 장치에서는 `millis()` 기반 상태 머신으로 바꾸어 Serial 명령과 LED 제어를 동시에 처리한다.
- 한글 좌표는 동적 문자열 대신 `const` 테이블과 플래시 메모리를 사용한다.
- Linux와 Arduino 통신에는 명령, 길이, 체크섬을 포함하여 잘못된 패킷을 거부한다.
- RTC 읽기 실패 시 마지막 정상 시각을 표시하고 오류를 Serial로 전송한다.

## 13. 실제 회로와 시뮬레이션의 차이

| 항목 | 시뮬레이션 | 실제 회로 |
| --- | --- | --- |
| 부품 오차 | 대부분 이상적인 값 | 저항, LED 전압, RTC 오차가 존재 |
| 접촉 불량 | 거의 없음 | 브레드보드와 점퍼선에서 발생 가능 |
| 전원 노이즈 | 단순화됨 | LED 전환과 USB 전원 품질의 영향을 받음 |
| 한글 문자판 | 논리적 LED 위치만 확인 | 격벽, 확산판, 빛 번짐이 중요 |
| RTC 백업 | 완전한 재현이 어려움 | 코인전지 방전과 장기 오차 확인 필요 |
| Linux USB 연결 | Tinkercad에서 직접 재현하기 어려움 | 장치명, 권한, 재연결 처리가 필요 |

시뮬레이션 통과는 실제 제작 성공을 보장하지 않는다. 실제 장치에서는 전류, 온도, 전원 재인가, USB 재연결, 24시간 이상 연속 동작을 추가로 시험해야 한다.

시뮬레이션을 직접 수행한 뒤 다음 파일을 저장한다.

```text
arduino/
├── 아두이노_시뮬레이션.md
└── assets/
    ├── simulation_sos_fritzing.png
    ├── simulation_sos_tinkercad.png
    ├── simulation_clock_fritzing.png
    ├── simulation_clock_serial.png
    └── simulation_clock_wokwi.png
```

문서에 이미지를 추가할 때에는 다음 형식을 사용한다.

```markdown
![S.O.S Tinkercad 시뮬레이션](./assets/simulation_sos_tinkercad.png)
```

영상은 저장소 용량을 고려해 링크로 첨부하거나 짧게 압축하고, 개인 계정 정보가 화면에 보이지 않도록 잘라낸다.

## 16. 결론

LED S.O.S 회로는 Arduino D13, 220Ω 저항, LED만으로 구성할 수 있으며 예상 전류는 약 13.6mA이다. 코드상 점 200ms, 선 600ms의 모스 부호가 반복된다.

한글시계는 DS1307에서 읽은 시각을 Arduino가 한글 좌표로 바꾸고 MAX7219를 통해 8×8 LED 문자판에 표시한다. 기존 코드는 전체 구조를 보여 주는 축약본이므로 모든 시·분 좌표를 구현해야 완성된다.

Fritzing은 배선과 회로 문서화, library.io는 부품·풋프린트 검토, Tinkercad Circuits는 Arduino 코드 시뮬레이션에 사용하는 것이 적절하다. 임베디드 리눅스 요구사항은 Raspberry Pi가 상위 제어와 기록을 담당하고 Arduino가 실시간 I/O를 담당하는 방식으로 충족한다.
